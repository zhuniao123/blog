## 🧰 技能工厂的诞生

[markdown-viewer/skills](https://github.com/markdown-viewer/skills) 最值得尊重的地方，不是它会画多少图，而是它把“画图这件事”变成了可分发、可组合、可迁移的技能单元。

它把常见需求拆得很干净：
- 数据图表交给 vega
- 信息编排交给 infocard 和 infographic
- 架构、流程、网络、安全交给 PlantUML 系列技能

这不是花活，这是工程化。你给 AI 一套稳定的边界条件，AI 才有机会少犯低级错误。

## 🏭 规模化的代价

当一个仓库把 14 类技能、5 种渲染引擎、成体系的 code fence 规范全部打包好，它自然会形成一种“即插即用的生产力幻觉”。

你会以为自己在做设计，实际上你在调模板。
你会以为自己在做架构，实际上你在选图标库。

这不是仓库的问题，这是使用者的问题。工具链越强，人越容易把判断力托管给默认值。

## 🧠 观测者结论

这个项目的价值非常明确：
- 它降低了 AI 生成可视化内容的失败率
- 它让跨代理复用技能成为现实
- 它把“文档即界面”的体验往前推了一步

真正需要警惕的点同样明确：
- 技能包能标准化表达，不能替代思考
- 图看起来越专业，越要追问结构是否真实
- 你在复制模板时，也在复制偏见

如果你把它当作一把瑞士军刀，它会提升你的表达密度。
如果你把它当作自动驾驶，它会放大你的认知惰性。
