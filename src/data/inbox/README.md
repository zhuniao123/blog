# 📥 极客投喂收件箱 (Observation Inbox)

## 📖 如何投喂素材？
1. 直接在此目录下新建任何文件（.txt, .md）。
2. 内容可以是：
   - 一个 GitHub 链接。
   - 一句吐槽。
   - 一段你觉得有意思的新闻摘要。
   - 甚至只是一个关键词。

## ⚙️ 处理逻辑
当你对我说“清空收件箱”时，我会：
1. 读取此处所有非 README 的文件。
2. 逐条解析链接或条目，执行“一条链接一篇文”，不再合并为一篇总论。
3. 每篇文章使用【边缘观测者】的“诙谐+建设性”模式：
   - 先讲清这个 skill 是什么、解决什么问题。
   - 再给一个轻松好记的比喻（帮助读者记住）。
   - 最后给出适用场景和踩坑提醒（避免为毒舌而毒舌）。
4. 自动归档到 blog 目录并推送到 GitHub。
5. 清空此目录。

## 🧭 写作约束（默认启用）
1. 目标优先级：帮助理解 > 帮助记忆 > 风格表达。
2. 语气要求：可以幽默，但不做人身攻击，不做情绪化贬损。
3. 结构模板：
   - 这是什么（30 秒看懂）
   - 为什么值得记（一句比喻）
   - 什么时候用（2-3 个场景）
   - 别怎么用（1-2 个常见误区）
4. 若收件箱中有 N 条链接，默认产出 N 篇文章（可在指令中指定只产出前 K 篇）。

## 🗂️ 每日投喂格式（推荐）
你每天可以新增一个文件，例如：`2026-04-23-chinese.md`。

建议每条素材使用下面结构：

```md
标题：
来源链接：
提示词原文：
图片链接：
场景标签：（如 小红书 / 公众号封面 / 电商详情图）
你想要的输出角度：（如 教程拆解 / 轻吐槽 / 案例复盘）
```

## ✅ 你需要做的事（最小清单）
1. 提供可复制的提示词文本（不要只放截图）。
2. 提供至少 1 个可访问图片链接（最好 2-3 个）。
3. 标注发布场景（我会据此调整文章结构和语气）。
4. 对我说“清空收件箱”，我会按“一条素材一篇文”处理并推送。
